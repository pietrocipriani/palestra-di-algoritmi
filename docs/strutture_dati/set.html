Palestra di Algoritmi

Insiemi

L'insieme è una collezione di elementi unici (non duplicati).

In C++ esistono due implementazioni: set e unordered_set.
Le implementazioni hanno un'interfaccia comune, cambiano alcune proprietà e i costi delle operazioni.

Esistono anche le varianti multiset e unordered_multset per collezioni che ammettono elementi duplicati.

Per poter scorrere gli elementi è necessario usare il costrutto foreach:

for (const auto& elemento : insieme) {
    // fai qualcosa con `elemento`.

    // `elemento` è `const`: se si modifica il valore la struttura dato potrebbe perdere consistenza.
    // Per esempio si vedano i BST (alberi binari di ricerca).

    // Un elemento, per essere modificato, deve essere rimosso, modificato e riaggiunto.
}

Interfaccia

  • Capacità:
    • empty(): ritorna true se l'insieme è vuoto, false altrimenti.
    • size(): ritorna il numero di elementi dell'insieme.
  • Modificatori:
    • clear(): svuota l'insieme.
    • insert(TIPO valore): inserisce valore all'interno dell'insieme.
  • Ricerca:
    • find(TIPO valore): ritorna l'iteratore associato all'elemento valore, ::end() se l'elemento non è contenuto nell'insieme.

Set

In set<TIPO> gli elementi sono memorizzati in ordine.
Di default l'ordine è crescente, tuttavia è possibile utilizzare l'ordine decrescente:

set<TIPO, greater<TIPO>> insieme;

Modifiche e ricerca hanno complessità logaritmica.

Unordered Set

L'unordered_set<TIPO> è un insieme non ordinato.
Ciò permette di ottenere complessità costante di accesso e modifica.

I coefficienti moltiplicativi dell'unordered_set sono abbastanza elevati.
