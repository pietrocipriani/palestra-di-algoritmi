Palestra di Algoritmi

Grafi

I grafi sono delle strutture dati molto versatili che possono essere impiegate per modellizzare una grande varietà di problemi.

Il grafo può essere visto come una serie di nodi collegati da archi.

Il grafo si distingue dalle altre strutture dati proprio per la sua capacità di mettere in relazione i dati tramite dei collegamenti.
I collegamenti possono essere orientati (con l'orientamento indicato da una freccia) o non orientati.

Implementazioni

Il C++ NON definisce un container standard per i grafi. Devono essere implementati a mano.

Eventuali metadati associati al nodo stesso possono essere memorizzati in un vettore dei nodi.

Quanto segue può risultare un po "astratto", avrete modo di vedere negli esercizi degli esempi più concreti.

Esistono due modi principali per costruire un grafo: tramite matrice di adiacenza e lista di adiacenza.

Matrice di adiacenza

Questo modo di rappresentare un grafo è poco utilizzato dato che, in grafi "sparsi" (ossia con pochi collegamenti), è poco efficiente.

L'idea è che, dati N nodi indicizzati, venga costruita una matrice N·N di bool rappresentante l'esistenza (true) o meno (false) del collegamento fra il nodo x e il nodo y.

// Numero di nodi
int N = ...;

// Matrice N·N di adiacenza rappresentante un grafo senza archi.
vector<vector<bool>> matrice_di_adiacenza(N, vector<bool>(N, false));

// Aggiungo un arco dal nodo `5` al nodo `10` (5 -> 10).
matrice_di_adiacenza[5][10] = true;
// Aggiungo un arco dal nodo `10` al nodo `5` (10 -> 5): non è la stessa cosa dell'arco 5 -> 10!
matrice_di_adiacenza[10][5] = true;

Lista di adiacenza

L'idea è che ogni nodo abbia una lista in cui memorizza i suoi "vicini".

// Numero di nodi
int N = ...;

// Vettore di dimensione N (1 elemento per ogni nodo) contenente la lista di vicini.
// Contiene `int`: l'indice del nodo vicino.
// Non inizializziamo la lista, non ne conosciamo la dimensione: inizialmente è vuota.
vector<vector<int>> lista_di_adiacenza(N);

// Aggiungo `10` come vicino di `5` (5 -> 10).
lista_di_adiacenza[5].push_back(10);
// Aggiungo `5` come vicino di `10` (10 -> 5).
lista_di_adiacenza[10].push_back(5);

// ATTENZIONE: aggiungere nuovamente un arco crea un NUOVO collegamento duplicato. Questo non era un problema con la matrice di adiacenza.
// Il grafo contiene due archi 5 -> 10. Ciò solitamente è un errore che può compromettere la correttezza degli algoritmi.
lista_di_adiacenza[5].push_back(10);

Esempio

Si propone un esempio completo di programma in grado di costruire un grafo orientato a partire da un "input.txt" dove:

  • La prima riga contiene N e M: rispettivamente il numero di nodi e archi.
  • La successiva riga contiene N informazioni relative ai vari nodi. In questo caso il colore (numero intero).
  • Le successive M righe contengono da e a. Dichiarano cioè l'esistenza di un arco da da verso a.

Un esempio di input è il seguente:

5 2
2 0 0 1 2
0 3
2 5

#include <bits/stdc++.h>
using namespace std;

// Nel caso decidessimo di cambiare il modo in cui implementiamo la lista di adiacenza, è "sufficiente" modificarla qui.
using ListaAdiacenza = vector<int>;

// Un nodo, contenente il proprio colore e la lista dei vicini.
struct Nodo {
    int colore;
    ListaAdiacenza vicini;
};

int main() {
    ifstream in("input.txt");
    ofstream out("output.txt");

    // Leggiamo N e M.
    int N, M; in >> N >> M;

    // Associamo ad ogni nodo le sue informazioni.
    // Il grafo è implementato come LISTA DI ADIACENZA.
    vector<Nodo> grafo(N);

    // Inizializziamo i nodi (colore).
    // In alcuni problemi questa parte è OPZIONALE, dato che inizialmente non è associata nessuna informazione ai nodi.
    // Scorriamo PER RIFERIMENTO (IMPORTANTE: vogliamo modificare il colore) i nodi.
    for (Nodo& nodo : grafo) {
        in >> nodo.colore;
    }

    // Leggiamo tutti gli `M` archi.
    for (int i = 0; i < M; ++i) {
        // Leggiamo `da` e `a`.
        int da, a; in >> da >> a;

        // Di solito abbiamo la garanzia che l'arco `da->a` non sia già stato inserito.
        // Quindi possiamo aggiungerlo senza troppi problemi.
        grafo[da].vicini.push_back(a);

        // Nel caso che il grafo sia NON ORIENTATO: dobbiamo aggiungere anche l'arco inverso.
        // grafo[a].vicini.push_back(da);
    }

    // TODO: facciamo qualcosa con il grafo.
    risolvi_problema(grafo);

    in.close(); out.close();
}
