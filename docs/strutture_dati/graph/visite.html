Palestra di Algoritmi

Visite nei grafi

Con visita di un grafo si intende la visita (accesso) dei nodi.

L'importanza dei grafi sta nei collegamenti fra i nodi, quindi la visita dei nodi in ordine di indice NON è particolarmente utile.
L'idea è quella di "navigare" il grafo, partendo da un nodo e visitando pian piano i vicini, i vicini dei vicini e così via.

Esistono due tecniche "standard" per visitare i grafi: DFS e BFS.
Queste due tecniche hanno proprietà differenti che possono essere sfruttate in base al problema.

Nei cosiddetti grafi NON connessi, non è possibile raggiungere tutti i nodi partendo da un singolo nodo.
Se il problema richiede di visitare tutti i nodi è necessario far ripartire la visita da un altro nodo.

DFS

L'acronimo sta per depth first search ossia ricerca con priorità alla profondità.

La visita parte da un nodo specificato.
Si sfrutta la programmazione ricorsiva: similarmente al backtracking prima si prova a visitare seguendo una certa direzione, poi si torna indietro e si provano altre strade.

In quanto implementazione ricorsiva bisogna stare attenti allo stack overflow.

Si consiglia la visione di questo simulatore per visualizzare il funzionamento della DFS.

struct Nodo {
    // Flag per stabilire se siamo già arrivati a questo nodo.
    // Se si effettuano più visite è necessario RESETTARE questo valore a `false`.
    // Alcune volte può essere più comodo creare un `vector<bool> visitato(N, false)` per ovviare a questo problema.
    bool visitato = false;

    ListaAdiacenza vicini;
};

// Il grafo. Inizializzato.
vector<Nodo> grafo = ...;

// `posizione` è il nodo in cui ci troviamo.
// Le visite possono servire per ricavare qualche "informazione". La DFS può ritornare un qualcosa di tipo `RET`.
RET dfs(int posizione) {
    // Segnamo immediatamente il nodo come visitato.
    grafo[posizione].visitato = true;

    // TODO: faccio qualcosa con questo nodo.

    // Provo a saltare sui nodi vicini.
    for (int vicino : grafo[posizione].vicini) {
        // Se il nodo vicino è già stato visitato allora lo ignoro.
        if (grafo[vicino].visitato) continue;

        // Salto sul vicino.
        dfs(vicino);
    }

    // TODO: faccio qualcosa con le informazioni ottenute dai vicini.

    // NON ripristino `visitato`: l'idea è che non ci sia più bisogno di ritornare su questo nodo.
}

BFS

L'acronimo sta per breadth first search ossia ricerca con priorità alla larghezza.

Si parte da un nodo ben preciso (potenzialmente non è una limitazione).
Si visitano prima tutti i nodi vicini di "primo ordine", poi si passa ai vicini di "secondo ordine" (vicini dei vicini) e così via.
Si impiega una coda (queue) per "consumare" i nodi pian piano che arrivano.

Si consiglia la visione di questo simulatore per visualizzare il funzionamento della BFS.

struct Nodo {
    // Flag per stabilire se siamo già arrivati a questo nodo.
    // Se si effettuano più visite è necessario RESETTARE questo valore a `false`.
    // Alcune volte può essere più comodo creare un `vector<bool> visitato(N, false)` per ovviare a questo problema.
    bool visitato = false;

    ListaAdiacenza vicini;
};

// Il grafo. Inizializzato.
vector<Nodo> grafo = ...;

// `posizione` è il nodo di partenza.
// Le visite possono servire per ricavare qualche "informazione". La BFS può ritornare un qualcosa di tipo `RET`.
RET bfs(int partenza) {
    // La lista dei nodi in coda.
    queue<int> coda;

    // Inseriamo il nodo di `partenza`: è in lista di attesa per essere visitato.
    coda.push(partenza);
    
    // Segnamo immediatamente il nodo come visitato.
    // In realtà non lo abbiamo ancora visitato, ma, coerentemente con l'implementazione successiva, segnamo come `visitato` i nodi non appena li inseriamo in coda.
    grafo[partenza].visitato = true;

    // Finchè la coda non diventa vuota continuiamo a visitare i nodi in attesa.
    while (!coda.empty()) {
        // Analizzo il primo nodo in coda.
        int nodo = coda.front();
        // Rimuoviamo il nodo che stiamo analizzando dalla coda.
        coda.pop();

        // TODO: faccio qualcosa con questo nodo.

        for (int vicino : grafo[nodo].vicini) {
            // Se il vicino è già stato visitato (inserito in coda) lo ignoriamo.
            if (grafo[vicino].visitato) continue;
            
            // Inserisco il vicino in coda.
            coda.push(vicino);
            
            // L'abbiamo inserito in coda: lo segno come visitato.
            grafo[vicino].visitato = true;
        }

        // Al contrario della DFS non possono fare nulla con le informazioni dei vicini:
        // i vicini non sono ancora stati realmente visitati!
    }
}
