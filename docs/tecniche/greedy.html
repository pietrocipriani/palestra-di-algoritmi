Palestra di Algoritmi

Greedy

La traduzione di greedy è ingordo.

Con il backtracking abbiamo visto come sia possibile trovare la migliore risposta semplicemente andando a provare tutte le scelte possibili.
In alcuni casi è possibile "ad occhio" vedere che alcune scelte sono probabilmente migliori di altre.

L'approccio greedy effettua una decisione "ingorda", nel senso che cerca di massimizzare il "risultato parziale".
Se si riesce a dimostrarne la correttezza a livello globale allora l'algoritmo è completo.

Alcune volte la scelta ingorda non comporta una ottimizzazione globale della soluzione.
In alcuni problemi non è richiesto di avere LA soluzione, bensì il punteggio viene dato in base alla qualità della soluzione fornita.
L'approccio greedy può consentire di ottenere una buona soluzione.

L'approccio greedy può inoltre essere impiegato per "pilotare" l'esplorazione di un algoritmo di backtracking, in modo da visitare le soluzioni "più buone" per prime.
In certi algoritmi ciò permette di scartare a priori determinate soluzioni una volta che abbiamo stabilito un limite inferiore/superiore al punteggio.

Esempi

Il problema turni proposto in day1 aveva soluzione greedy.

#include <bits/stdc++.h>
using namespace std;

// Struttura rappresentante un intervallo di tempo.
struct Intervallo {
  int inizio, fine;
};

/**
 * Soluzione greedy al problema.
 * @param intervalli La lista ORDINATA di intervalli.
 * @param K Il numero di giorni da coprire.
 * @return Il numero minimo di persone da assumere.
 */
int solve(const vector<Intervallo>& intervalli, int K) {
  // Il primo tempo non ancora coperto.
  int inizio = 0;
  // Il tempo massimo che possiamo coprire con gli intervalli già valutati.
  int fine = 0;

  // Il numero di persone che devono essere assunte.
  int conteggio = 0;

  for (const Intervallo& intervallo : intervalli) {
    if (intervallo.inizio > inizio) {
      // Sono finiti gli intervalli che ci permettono di dare continuità alla copertura.
      // Vediamo i precedenti fino a dove ci permettono di coprire.
      
      // Dovremo iniziare a ricercare dal tempo appena successivo a quello di fine già trovato.
      inizio = fine + 1;

      // Abbiamo assunto un'altra persona
      ++conteggio;
    }

    // Scelta GREEDY: `fine` deve essere un massimo.
    // Non è detto che la soluzione ottima debba avere per forza il massimo. Tuttavia:
    // - Se non contiene il massimo possiamo provare a sostituire il primo intervallo con il massimo ed ottenere una soluzione ottima valida.
    // - La ricerca dei successivi intervalli è un sottoproblema del tutto analogo, ma con tempo di inizio differente. Possiamo ripetere lo stesso ragionamento.
    fine = max(fine, intervallo.fine);

    // Può tranquillamente succedere che finiscano gli intervalli senza che si esca dalla zona di copertura (ossia non si entra nell'if precedente).
    // Ciò può comportare che l'ultimo intervallo di copertura non venga conteggiato.
    // Gestiamo l'evento come caso particolare.
    // Ciò ci permette inoltre di terminare in anticipo.
    if (fine >= K - 1) {
      ++conteggio;
      break;
    }
  }
}

// Serve per il `sort` degli intervalli (secondo `inizio`).
// Si potevano tranquillamente usare delle `std::pair` senza dover ridefinire l'operatore `<`.
bool operator<(const Intervallo& i1, const Intervallo& i2) {
  // In caso di pareggio si potrebbe valutare la `fine`. Tuttavia per il nostro algoritmo non cambia nulla.
  return i1.inizio < i2.inizio;
}

int main() {
  ifstream in("input.txt");
  ofstream out("output.txt");
  
  // Numero di giorni
  int K; in >> K;

  // Numero di persone
  int N; in >> N;

  // Come detto nella teoria dei `vector`, è meglio fornire di già la dimensione.
  vector<Intervallo> intervalli(N);

  // Leggiamo tutti gli intervalli.
  for (Intervallo& intervallo : intervalli) {
    in >> intervallo.inizio >> intervallo.fine;
  }

  // Ordiniamo gli intervalli in modo da avere quelli che iniziano prima per primi.
  // È una proprietà che verrà sfruttata dal resto dell'algoritmo.
  sort(intervalli.begin(), intervalli.end());

  out << solve(intervalli, K);

  in.close(); out.close();
}
