Palestra di Algoritmi

Divide et Impera

Il divide et impera è una tecnica risolutiva ricorsiva che prevede di suddividere (divide) il problema in problemi uguali ma più piccoli, risolverli (impera) in maniera ricorsiva per poi combinare i risultati ottenuti dai singoli sottoproblemi per ottenere il risultato del problema più grande.

Ovviamente la tecnica non può essere applicata sempre e richiede inventiva dal momento che non sempre è straight-forward.

Esempi

Ricerca binaria / dicotomica

Vogliamo scrivere un'algoritmo che ci dica se un elemento si trova all'interno di una sequenza ordinata.

Una soluzione può prevedere sicuramente di scorrere tutti gli elementi:

bool contains(const vector<int>& sequenza, int elemento_da_trovare) {
    for (int elemento : sequenza) {
        // Se abbiamo trovato l'elemento allora l'elemento è contenuto all'interno della sequenza: sequenza contains elemento_da_trovare = true.
        if (elemento == elemento_da_trovare) return true;
    }
    // Nessun elemento è uguale a `elemento_da_trovare`: sequenza contains elemento_da_trovare = true.
    return false;
}

Questa soluzione è lineare nel caso pessimo (e medio).

Possiamo sfruttare il fatto che la sequenza sia ordinata a nostro vantaggio: possiamo controllare un elemento a caso:

  • se è uguale all'elemento da trovare allora abbiamo concluso.
  • se è maggiore dell'elemento da trovare allora questo si troverà alla sua "sinistra" (ordinamento crescente).
  • se è minore allore si troverà alla sua "destra".

Dal momento che vogliamo ottimizzare il caso medio/pessimo, ha senso fare in modo che la partizione "sinistra" e "destra" abbiano la stessa dimensione:
con una partizione sbilanciata otteniamo una maggiore probabilità che l'elemento da trovare si trovi nella partizione più grande, e più la partizione è grande più è difficile trovare l'elemento.
Andremo dunque a verificare l'elemento che si trova a metà della partizione che stiamo valutando.

Esempio: vogliamo ricercare il numero 3 all'interno della sequenza 1 2 3 4 5 6 7 8 9.

  • controlliamo il 5 che si trova a metà: ci dice che dobbiamo ricercare il 3 in 1 2 3 4 dal momento che è più grande.
  • controlliamo il 2 (dipende dall'implementazione): ci dice che dobbiamo ricercare il 3 in 3 4.
  • controlliamo il 3 che effettivamente ci dice che l'elemento è presente nella sequenza.

Esempio: vogliamo ricercare il numero 3 all'interno della sequenza 1 2 4 5 6 7 8 9.

  • controlliamo il 5: 1 2 4.
  • controlliamo il 2: 4.
  • controlliamo il 4: otteniamo una partizione vuota che ci dice che l'elemento non esiste.

La complessità di questo algoritmo è logaritmica nel caso pessimo!
La dimensione della partizione viene dimezzata ad ogni passaggio, quindi anche con una sequenza di \(2^{100}\) elementi, dopo soli \(100\) passaggi avremo individuato l'elemento.
Con l'altro algoritmo ci sarebbero voluti (al peggio) \(2^{100}\) passaggi.

Viene presentata una possibile implementazione:

// `from` inclusivo, `to` esclusivo.
bool contains(const vector<int>& sequenza, int from, int to, int elemento_da_trovare) {
    // Abbiamo una sequenza di 0 elementi: false.
    if (from >= to) return false;

    // Indice del valore che si trova in mezzo fra `from` e `to`.
    // ps: si poteva fare (from + to) / 2.
    // Tuttavia questa scrittura permette di non avere overflow.
    // È inoltre concettualmente più corretta quando si ha a che fare con iteratori.
    int mid = from + (to - from) / 2;

    if (sequenza.at(mid) == elemento_da_trovare) {
        // Abbiamo trovato l'elemento.
        return true;
    } else if (sequenza.at(mid) > elemento_da_trovare) {
        // L'elemento sta a "sinistra".
        return contains(sequenza, from, mid, elemento_da_trovare);
    } else /* if (sequenza.at(mid) < elemento_da_trovare) */ {
        // L'elemento sta a "destra".
        return contains(sequenza, mid + 1, to, elemento_da_trovare);
    }
}

Quick Sort

Adesso che abbiamo un buon algoritmo per ricercare elementi di una sequenza ordinata abbiamo bisogno di un buon algoritmo di ordinamento.

Facciamo finta di sapere il valore mediano fra tutti gli elementi della sequenza.

  • Tutti gli elementi più piccoli del mediano li mettiamo all'inizio.
  • Tutti gli elementi più grandi del mediano li mettiamo alla fine.

Dopodichè procediamo nella stessa maniera sulle due partizioni individuate.

L'implementazione "base" del Quick Sort non conosce il mediano e si limita a prendere uno degli estremi come valore "a caso" che in media funge da mediano e che è detto pivot.

La complessità ottima / media del Quick Sort è quasi-lineare (N·log(N)).
Nel caso pessimo, tuttavia, è quadratica. Il caso pessimo corrisponde a sequenze già quasi ordinate, dove la scelta degli estremi porta ad ottenere partizioni sbilanciate.

Esempio

Vogliamo ordinare la sequenza 6 10 3 0 2 4 8 5 9 1.

  • Scegliamo come pivot l'1. La scelta non è delle migliori: 6 10 3 0 2 4 8 5 9 + 1 + \(\emptyset\).
  • Scegliendo il mediano 5 si otteneva: 3 0 2 4 + 5 + 6 10 8 9.
    • 3 0 2 4: il pivot è 2: 0 + 2 + 3 4.
      • 3 4: il pivot è 3: \(\emptyset\) + 3 + 4. Sottosequenza ordinata: 3 4.
      • Otteniamo la sottosequenza ordinata 0 2 3 4.
    • 6 10 8 9: pivot 8: 6 + 8 + 10 9.
      • 10 9. Sottosequenza ordinata 9 10.
      • Sottosequenza ordinata: 6 8 9 10.
    • Sottosequenza ordinata: 0 2 3 4 5 6 8 9 10: abbiamo concluso.

Merge Sort

Altro algoritmo di ordinamento.

L'idea è sempre quella di dividere gli elementi, tuttavia in questo caso prima applichiamo il divide.
Vogliamo avere due sottosequenze già ordinate per poi andare a farne il merge che è un'operazione poco costosa quando le sequenze sono già ordinate.

In questo caso la complessità è sempre quasi-lineare.

Esempio

Vogliamo ordinare la sequenza 6 10 3 0 2 4 8 5 9 1.

  • Ordiniamo 6 10 3 0 2.
    • Ordiniamo 6 10 3.
      • Ordiniamo 6 10.
        • Ordiniamo 6.
        • Ordiniamo 10.
        • Combiniamo ed otteniamo 6 10.
      • Ordiniamo 3.
      • Combiniamo ed otteniamo 3 6 10.
    • Ordiniamo 0 2.
      • Ordiniamo 0.
      • Ordiniamo 2.
      • Otteniamo 0 2.
    • Otteniamo 0 2 3 6 10.
  • Ordiniamo 4 8 5 9 1.
    • Ordiniamo 4 8 5.
      • Ordiniamo 4 8.
        • Ordiniamo 4.
        • Ordiniamo 8.
        • Otteniamo 4 8.
      • Ordiniamo 5.
      • Otteniamo 4 5 8.
    • Ordiniamo 9 1.
      • Ordiniamo 9.
      • Ordiniamo 1.
      • Otteniamo 1 9.
    • Otteniamo 1 4 5 8 9.
  • Otteniamo 0 1 2 3 4 5 6 8 9 10.
