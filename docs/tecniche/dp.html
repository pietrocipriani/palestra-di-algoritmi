Palestra di Algoritmi

Programmazione dinamica

L'obiettivo della programmazione dinamica è lo stesso della memoization, ma con approccio "opposto".

Confronto

La memoization è migliore della programmazione dinamica quando vengono compilate solo poche celle della cache, e non è possibile prevedere quali.
La programmazione dinamica è iterativa e pertanto non presenta l'overhead della ricorsione.
La programmazione dinamica permette un migliore controllo della cache, permettendo ottimizzazioni di memoria.
La memoization è tendenzialmente più semplice da implementare, partendo da una soluzione backtracking.

Esempio - Domino

Proseguendo dalla memoization:

// Ci creiamo una cache di dimensione opportuna (dato dalle constraints nel pdf, oppure la N letta da input).
// Inizializziamo a 0, se il valore è 0 allora sicuramente non lo abbiamo calcolato (Fibonacci ha valori > 0).
vector<int> cache(N + 1, 0);

int domino(int N) {
    // Se lo abbiamo già calcolato allora restituiamo il valore trovato.
    if (cache[N] > 0) return cache[N];

    // Altrimenti proseguiamo con l'algoritmo precedente.
    
    if (N <= 1) return cache[N] = 1;

    // Memorizziamo il valore in modo che sia disponibile per chiamate successive.
    return cache[N] = domino(N - 1) + domino(N - 2);
}

È possibile osservare che domino(N) dipende da alcuni domino(M) con M < N.
Possiamo dunque compilare la cache partendo da 0, in modo da avere i valori necessari a calcolare i nuovi risultati già precalcolati.
Se abbiamo già calcolato i valori, una chiamata a domino(i) restituisce direttamente cache[i], possiamo dunque sostituire tutte le chiamate domino(i) con cache[i].

vector<int> cache(N + 1, 0);

// Casi base (ps: potenzialmente N < 1, assumiamo però che non sia così per leggibilità).
cache[0] = cache[1] = 1;

for (int i = 2; i <= N; ++i) {
    // Implementazione equivalente a quella della funzione ricorsiva.
    cache[i] = cache[i - 1] + cache[i - 2];
}

La soluzione domino(N) si troverà in cache[N].

Non ha senso salvare tutti i valori, ci servono solo i due precedenti:

// `prev2` rappresenta `cache[i - 2]`, inizialmente `i == 2` quindi `prev2 = cache[0]`, inizializzato come caso base.
// `prev1` rappresenta `cache[i - 1]`, inizialmente `i == 2` quindi 'prev1 = cache[1]', inizializzato come caso base.
int prev2 = 1, prev1 = 1;

for (int i = 2; i <= N; ++i) {
    int nuovo_valore = prev2 + prev1;
    // l'attuale `prev1` diventerà `prev2` all'iterazione successiva.
    prev2 = prev1;
    // l'attuale `nuovo_valore` diventerà `prev1` all'iterazione successiva.
    prev1 = nuovo_valore;
}

La soluzione domino(N) si troverà in prev, ossia il valore precedente dell'eventuale iterazione successiva.

Il consumo di memoria è costante.
Con la memoization risulta molto più difficile fare un'ottimizzazione del genere.

Esempio - Sommelier

Proseguendo dalla memoization:

vector<int> vini = ...;

// Il primo indice è relativo a `i`.
// Il secondo indice è relativo a `minima_gradazione`.
// `i` può spaziare da 0 a N con N escluso (N valori).
// `minima_gradazione` può spaziare da 0 a MAX_GRADAZIONE inclusi (MAX_GRADAZIONE+1 valori).
// Inizializziamo a -1 perchè 0 è un valore valido di risposta.
vector<vector<int>> cache(N, vector<int>(MAX_GRADAZIONE + 1, -1));

int sommelier(int i = 0, int minima_gradazione = 0) {
  if (i >= vini.size()) return 0;

  int& result = cache[i][minima_gradazione];
  // Se il valore è già stato calcolato allora restituiamo il valore.
  if (result >= 0) return result;

  // Altrimenti proseguiamo con l'algoritmo.

  int gradazione = vini.at(i);

  if (gradazione < minima_gradazione) {
    // Salviamo il risultato.
    return result = sommelier(i + 1, minima_gradazione);
  } else {
    // Salviamo il risultato.
    return result = max(
      sommelier(i + 1, minima_gradazione),
      sommelier(i + 2, vino) + 1
    );
  }
}

È possibile osservare che sommelier(i, minima_gradazione) dipende da sommelier(j, gradazione) con j > i e minima_gradazione <= gradazione.

Abbiamo vari modi per compilare in maniera iterativa la cache.
Possiamo per esempio ciclare esternamente con i in modo da avere già pronti tutti gli indici successivi.
In questa maniera non abbiamo vincoli sulla minima_gradazione.
Altrimenti possiamo ciclare esternamente con la minima_gradazione, in ordine decrescente, per poi ciclare internamente con gli i anche in questo caso in ordine decrescente.

Scegliamo la prima strategia che ci permette inoltre di effettuare ottimizzazioni di memoria, dal momento che ci interessano solamente gli i+1 e gli i+2.

vector<int> vini = ...;

// Cache simile alla soluzione con memoization.
// Non abbiamo la necessità di inizializzare con un valore significativo:
// andiamo a compilare celle che sappiamo non essere già state compilate.
// `N + 2` in modo da poter gestire i casi base `i >= N`.
vector<vector<int>> cache(N + 2, vector<int>(MAX_GRADAZIONE + 1));

// Dobbiamo accedere alle celle corrispondenti ai vini successivi (i+1 e i+2). Abbiamo la necessità che siano già state compilate.
// Iteriamo quindi a ritroso.
for (int i = N - 1; i >= 0; --i) {
  // In realtà non abbiamo nessuna garanzia che siano necessari TUTTI i valori di gradazione.
  // Potenzialmente la memoization è meglio.
  for (int minima_gradazione = 0; minima_gradazione <= MAX_GRADAZIONE; ++minima_gradazione) {
    // Stesso codice della versione ricorsiva (memoization), con le chiamate a funzione sostituite dall'accesso in cache.
      
    // `i + 1` e `i + 2` possono rilanciare a `N` e `N + 1` che non sono indici validi per i vini.
    // Siccome per `i >= N` la risposta è 0, abbiamo creato 2 `vector` inizializzati a 0 in più; in modo da gestire `N` e `N + 1`.
    int vino = vini.at(i);
    
    int& result = cache.at(i).at(minima_gradazione);
    if (vino < minima_gradazione) result = cache.at(i + 1).at(minima_gradazione); // sommelier(i + 1, minima_gradazione)
    else result = max(
      cache.at(i + 1).at(minima_gradazione),  // sommelier(i + 1, minima_gradazione)
      cache.at(i + 2).at(vino) + 1            // sommelier(i + 2, vino)
    );
  }
}

La soluzione del problema sommelier(0, 0) si troverà in cache.at(0).at(0).

Come anticipato era possibile ottimizzare la memoria:

vector<int> vini = ...;

// Cache ridotta, ci servono solo due righe: `i + 1` e `i + 2`.
// I casi base (`N` e `N + 1`) sono inizializzati a 0.
vector<vector<int>> cache(2, vector<int>(MAX_GRADAZIONE + 1, 0));

// Iteriamo sempre a ritroso.
for (int i = N - 1; i >= 0; --i) {
  // Potenzialmente potevamo creare un `vector` di appoggio da poi assegnare a `cache[0]`.
  // Ps: l'assegnamento "efficiente" va fatto con `std::move`.
  // Interando in avanti la gradazione possiamo sostituire "in place".
  // Sicuramente faremo riferimento solo a gradazioni successive, quindi quelle devono rimanere invariate: vanno modificate dopo.
  for (int minima_gradazione = 0; minima_gradazione <= MAX_GRADAZIONE; ++minima_gradazione) {
    int vino = vini.at(i);
    
    // Questa volta non è un riferimento, in modo da gestire "con calma" la modifica in place senza effettuare errori.
    int result;
    if (vino < minima_gradazione) result = cache.at(0).at(minima_gradazione); // sommelier(i + 1, minima_gradazione)
    else result = max(
      cache.at(0).at(minima_gradazione),  // sommelier(i + 1, minima_gradazione)
      cache.at(1).at(vino) + 1            // sommelier(i + 2, vino)
    );
    // `i+1` -> `i+2`
    cache.at(1).at(minima_gradazione) = cache.at(0).at(minima_gradazione);
    // `i` -> `i+1`
    cache.at(0).at(minima_gradazione) = result; // sommelier(i, minima_gradazione)
  }
}

Essendo MAX_GRADAZIONE una costante, non abbiamo ridotto la classe di complessità della memoria che rimane lineare.
Altrimenti la classe di complessità passerebbe da \(\Theta\left(N\cdot\text{MAX_GRADAZIONE}\right)\) a \(\Theta\left(N\right)\).
Con \(N\) sufficientemente grande risparmiamo almeno il \(97\%\) di memoria.
